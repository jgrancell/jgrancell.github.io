My Passion Projects

I am regularly working on 2-3 main passion projects. These are multi-year projects that have caught my imagination, or are projects that I believe could be widely used if I decided to market them.

Metasync

Metasync helps simplify managing multiple application repositories by letting you commit your meta files into a template directory and keep them synced to any number of application repositories. Meta files include things like:

  • CI/CD files (.gitlab-ci.yml, .travis.yml, etc)
  • Language files (Rakefiles, Makefiles, Composer.json/Package.json and similar package manager files)
  • License files
  • Ignore files (gitignores, dockerignores, etc)

With metasync, you can version your template files in a repository and have your application repositories pinned to a specific version, or kept up to date on a live and ever-changing branch.

Metasync saves you time by letting you update a single file in your template, and then running a CLI command to update your application repository. You don't need to copy and paste or manually update any more. I personally use the tool on all of my PHP and Golang applications to keep ~10 files per repository in sync across ~15 repositories.

Raidkeeper

Raidkeeper is a tool that I wrote for my World of Warcraft guild. It lets us manage our raiding roster and keep track of our characters development over a raiding tier, saving us hours of time every week. Features include:

  • Roster management
  • Automatic fetching of roster character equipment, raid logs, and mythic plus scores.
  • Weekly tracking of target goals, such as M+ dungeon completion.
  • Integration with Discord to provide daily updates towards weekly goals.
  • Analysis of raid composition to determine class or spec deficits or missing buffs.

Raidkeeper saves us time, makes running a raid team less of a job, and helps me scratch an itch to build highly complex applications.

Turtleshell

Turtleshell was my first attempt to really dig into Go. Since I learn best from actually writing something, I decided to write a Linux shell. I know...ambitious. But it was and continues to be one of my favorite projects. I'm mostly done with writing Turtleshell, though I hope one day to come back to it and learn how to write a programming language through adding Turtlescript to it. Features of Turtleshell include:

  • The ability to run binaries on the system and interact with the filesystem.
  • A number of custom written shell builtins, such as "cd", "echo", etc.
  • The ability to read shell configurations from .turtlerc files.
  • The ability to customize the shell prompt, which updates in real time.
  • History saving, including optional timestamps.

As I said, I do hope one day to write Shellang, the Turtleshell shell scripting language. Learing to write a Lexical parser and other aspects of a programming language is something I am very keen to do, and Turtleshell provides an excellent home for that. Perhaps one day if/when I have time.
